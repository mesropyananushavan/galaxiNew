# PM Analysis Summary

## Conclusion
`galaxiOld` is the product baseline. `galaxiNew` is still foundation-stage. Migration should be parity-first.

## Priority areas
- auth and permissions
- shops and access scope
- cardholders/cards/card types
- checks and points
- gifts
- core reports

## Suggested sequence
1. foundation
2. core operations
3. transactions
4. rewards/services
5. reporting
6. secondary areas

## Main risks
- hidden legacy business rules
- underestimating scope
- incorrect domain normalization
- broken shop-level access
